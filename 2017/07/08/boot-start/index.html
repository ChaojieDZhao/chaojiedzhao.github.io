为什么lambda需要final
如上所示，如果在lambda表达式内部使用注释的代码，则会提示以下错误。 local variables referenced from a lambda expression must be final or effectively final 那么为什么lambda表达式内部引用外部局部变量需要声明为final呢？ lambda可以理解为本来就是一个方法，如右侧所示，本地变量是存储在栈帧的局部变量表的，是私有的。而方法的传参是值传递的，所以在内部遍历中对变量的操作是不会影响到外部局部变量的。 但是语义上来说我们常常把lambda stream和for循环看做一样。其实内部实现是不一样的。lambda可以看做是调用了一个新的方法，而for循环本身就是方法的一部分。 所以既然如此，那java就把外部变量直接声明为final类型，或者effectively final（不会再lambda内部对外部引用变量修改）。这样保证了语义的一致性。 参考链接
